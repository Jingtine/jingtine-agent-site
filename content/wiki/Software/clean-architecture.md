# Clean Architecture

Clean Architecture separates software into layers with clear
dependency rules, enabling independent development and testing.

## Layers

- **Entities**: Core business logic
- **Use Cases**: Application-specific rules
- **Interface Adapters**: Controllers, presenters, gateways
- **Frameworks**: External tools and libraries
