# 黑盒测试用例表

## BC01 · 首页完整信息展示

- **对应需求**：Req1
- **前置条件**：网站可访问
- **操作步骤**：
  1. 打开网站首页
  2. 检查浏览器标签页标题
  3. 检查页面可见文本
- **预期结果**：
  1. 标题栏显示 "Jingtine"
  2. 页面文本包含 "Jingtine"
  3. 页面文本包含 "Software Engineering"、"AI Agent"、"Product Innovation"
  4. 页面底部存在邮箱地址
- **用例类型**：正常

## BC02 · 五个导航入口可跳转

- **对应需求**：Req2
- **前置条件**：首页已加载
- **操作步骤**：
  1. 点击导航栏 "About" → 确认跳转到关于页
  2. 点击导航栏 "Blog" → 确认跳转到博客页
  3. 点击导航栏 "Research" → 确认跳转到论文页
  4. 点击导航栏 "Wiki" → 确认跳转到知识库页
  5. 从关于页点击 "Home" → 回到首页
- **预期结果**：每次点击都跳转到正确页面，当前页导航项有视觉高亮
- **用例类型**：正常

## BC03 · 博客文章列表展示

- **对应需求**：Req3
- **前置条件**：博客页面可访问
- **操作步骤**：
  1. 访问博客列表页
  2. 统计展示的文章卡片数量
  3. 检查每张卡片是否显示标题和日期
- **预期结果**：至少有 5 篇文章，每篇有标题、日期、摘要
- **用例类型**：正常

- **对应需求**：Req3
- **前置条件**：博客列表页已加载
- **操作步骤**：

  3. 检查页面是否展示文章正文
- **预期结果**：跳转到文章详情页，正文有可读内容
- **用例类型**：正常

## BC05 · RSS Feed 可解析

- **对应需求**：Req4
- **前置条件**：无
- **操作步骤**：
  1. 访问 `/feed.xml`
  2. 用 XML 解析器解析返回内容
  3. 检查是否包含 `<rss version="2.0">`
  4. 检查 `<item>` 数量是否 ≥ 5
- **预期结果**：feed.xml 是合法 XML，包含多篇文章
- **用例类型**：正常

---

## BC06 · RSS 文章按日期倒序

- **对应需求**：Req4
- **前置条件**：feed.xml 至少有 3 篇不同日期的文章
- **操作步骤**：
  1. 解析 feed.xml
  2. 提取每篇文章的日期
  3. 验证日期从新到旧排列
- **预期结果**：最新文章排在最前
- **用例类型**：边界

## BC08 · 外部链接安全性

- **对应需求**：Req5
- **前置条件**：Reader 页面已加载
- **操作步骤**：
  1. 检查页面中所有外部链接
  2. 验证链接使用 `https://`
  3. 验证外部链接不含内联事件（onclick/onload）
- **预期结果**：所有外部链接是 HTTPS，无内联事件
- **用例类型**：异常/高风险

## BC10 · 移动端窄屏页面不崩溃

- **对应需求**：Req1/Req2/Req8（跨页面一致性）
- **前置条件**：无
- **操作步骤**：
  1. 在 375px 宽度下访问首页
  2. 检查页面是否可正常滚动、内容可见
  3. 检查导航栏是否完整显示
  4. 同宽度访问博客页和 Wiki 页
- **预期结果**：所有页面在窄屏下内容可见，导航完整
- **用例类型**：边界

## BC11 · 页面在任何宽度下不出现横向滚动条

- **对应需求**：Req8
- **前置条件**：无
- **操作步骤**：
  1. 在 768px 宽度下访问首页、博客页、Wiki 页、Reader 页
  2. 检查页面是否无横向滚动（`scrollWidth ≤ innerWidth`）
- **预期结果**：所有页面无横向溢出
- **用例类型**：边界

- **对应需求**：Req3
- **前置条件**：模拟 `articles/index.json` 返回空数组
- **操作步骤**：
  1. 不修改源文件，仅从需求角度：验证博客页在没有文章时能否正常展示

- **预期结果**：页面不崩溃，显示空状态或友好提示
- **用例类型**：边界
