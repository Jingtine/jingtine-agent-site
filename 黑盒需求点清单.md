# 黑盒需求点清单

> 仅从用户需求和验收标准出发，不依赖内部实现文件。

---

## Req1 · 首页个人信息展示

**需求内容**：在浏览器中展示姓名、学校、专业、自我介绍、三个兴趣方向和邮箱。

**验收点**：
1. 首页能打开，浏览器标题正确
2. 页面包含姓名 "Jingtine"
3. 页面包含学校/专业信息
4. 页面包含自我介绍文字
5. 页面包含三个兴趣方向（Software Engineering / AI Agent / Product Innovation）
6. 页面底部存在邮箱联系方式

**用户可见检查方式**：打开首页，肉眼确认信息完整
**适合自动化测试**：是
**风险说明**：中 — 首页是最高访问量入口

---

## Req2 · 导航栏与多页面

**需求内容**：提供首页、关于我、博客、Research Papers 和 Wiki 五个入口。所有页面使用相同菜单。当前页面菜单高亮。页面之间能互相跳转。

**验收点**：
1. 所有入口在导航栏中可见
2. 所有页面共用相同结构的导航栏
3. 点击导航链接可跳转到对应页面
4. 当前页面对应的导航项有明显视觉高亮

**用户可见检查方式**：点击每个导航链接，确认跳转正确，当前页导航有高亮
**适合自动化测试**：是
**风险说明**：高 — 导航断裂用户无法浏览

---

## Req3 · Markdown 博客

**需求内容**：Markdown 源文件可以生成博客列表和文章页面。添加一篇 Markdown 后能生成对应网页。构建脚本重复运行不会重复追加内容。

**验收点**：
1. 博客列表页展示文章标题、日期、摘要
2. 点击文章可进入详情页查看完整内容
3. 文章数量与源文件数量一致
4. 文章列表按日期排列

**用户可见检查方式**：访问博客页面，查看文章列表，点击进入详情
**适合自动化测试**：是
**风险说明**：中 — 内容核心模块

---

## Req4 · RSS Feed

**需求内容**：根据博客 Markdown 元数据生成 RSS 2.0 的 feed.xml。每个 item 必须有 title、link、guid、pubDate、description。必须正确处理中文和 XML 特殊字符。RSS 按日期倒序。feed.xml 必须能被 XML 解析器验证。

**验收点**：
1. `feed.xml` 存在且能被 XML 解析器解析
2. 每个 item 包含必填字段
3. 文章按日期从新到旧排列
4. 中文和特殊字符被正确处理

**用户可见检查方式**：用 RSS 阅读器订阅 feed.xml，确认文章列表正确
**适合自动化测试**：是
**风险说明**：高 — XML 非法导致订阅端无法解析

---

## Req5 · RSS 阅读器 / 外部订阅

**需求内容**：展示从外部 RSS 源聚合的文章，来源清晰可辨，外部链接安全。

**验收点**：
1. 阅读器页面能展示多个来源
2. 每个来源可点击查看其文章列表
3. 外部文章链接可正常打开
4. 外部链接使用安全属性

**用户可见检查方式**：访问 Reader 页面，查看来源卡片，点击文章链接
**适合自动化测试**：是
**风险说明**：高 — 外部内容安全问题

---

## Req6 · Wiki 知识库

**需求内容**：提供结构化的知识库页面，支持分类浏览和搜索，可查看单篇文章详情。

**验收点**：
1. 知识库页面展示分类列表
2. 支持按分类筛选
3. 支持搜索功能
4. 点击文章可查看详细内容
5. 文章中的交叉引用链接可点击跳转

**用户可见检查方式**：访问 Wiki 页面，操作分类和搜索，点文章看详情
**适合自动化测试**：部分（搜索准确性需人工）
**风险说明**：中 — 数据量大

---

## Req7 · Research Papers

**需求内容**：展示研究论文列表，每篇包含标题、作者、摘要，可跳转到原文。

**验收点**：
1. 论文页面展示论文列表
2. 每篇论文显示作者和摘要
3. 点击可跳转到论文原文（arXiv）
4. 论文来源明确标识

**用户可见检查方式**：访问 Papers 页面，查看文章卡片
**适合自动化测试**：是
**风险说明**：低

---

## Req8 · 跨页面一致性

**需求内容**：所有页面使用统一的视觉风格、导航栏和页脚。

**验收点**：
1. 所有页面导航栏入口相同
2. 所有页面页脚结构相同
3. 所有页面颜色风格统一

**用户可见检查方式**：逐页浏览，对比导航栏和整体风格
**适合自动化测试**：是
**风险说明**：低
